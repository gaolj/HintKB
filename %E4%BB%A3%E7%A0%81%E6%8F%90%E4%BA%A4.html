内部资料 : 代码提交

This page last changed on 2013-10-14 by gaolj.

每次提交svn源代码时必须与库中版本比较差异，确定所修改内容没有任何问题后再提交

提交给svn的代码必须是正式代码，如果有任何临时性的代码（如：调试用、测试用等）则不充许提交

提交svn时必须要加上message，message要简明扼要

错误格式

提交者：高黎峻		// 冗余	
提交时间：2013-09-24	// 冗余
提交内容：		// 无用信息			
1、增加对https的支持	// 缺少jira号，查找对应问题难
2、修正http重定向出错问题

 

代码源文件提交，message正确格式如下：

XYXS-1242								// jira任务号
增加客户端在网吧多计费环境下上报PubwinIP与注册号等信息至Pubwin服务端的功能		// 任务全称
									// 如果有多个任务，则任务间要有换行（可选）
XYXS-1005		 
客户端菜单搜索游戏时，对多音字进行特殊处理

如果没有JIRA号的修改：

+ 增加文本控制		// 增加内容，格式为：++空格+内容
- 删除URL跳转功能		// 删除内容，格式为：-+空格+内容	
* 修改按钮的按下效果	// 修改内容，格式为：*+空格+内容

注：'//'是说明，不要加入message中；对于中间性文件版本，可以只有jira任务号而没有简单描述
